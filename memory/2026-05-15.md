## Bug Fix Session — Summarization 500 Error + ValidationError + ROC Year

### HTTP 500 — Unhandled Exception (Fixed)
- Root cause: `client.messages.create()` and `message.content[0].text` had no try/except; any `anthropic.APIError`, `IndexError`, `AttributeError` propagated as unhandled FastAPI 500
- Fix 1 (`services/summarization/__init__.py`): wrapped API call in `try/except anthropic.APIError → RuntimeError`; wrapped content extraction in `try/except (IndexError, AttributeError) → RuntimeError`
- Fix 2 (`apis/v1/routers/summary.py`): added `except Exception` catch-all with logger.exception + detail message; added `import logging`
- Also fixed: GET `/{document_id}/summary/{report_id}` was missing `source_type`, `attribution_prefix`, `forward_looking`, `rhetorical_risk_flag` (Phase 10E fields)

### ValidationError — MongoEngine enum choices (Fixed)
- Root cause: Claude returns arbitrary strings for choice-constrained fields; MongoEngine raises ValidationError on save
- Error example: `section_key.Value must be one of ['key_financials', ...]` for items 19/20/21
- Fix (`services/summarization/__init__.py`): added 6 `_VALID_*` sets; all choice fields normalized with fallback before `AIClaim()` construction
  - `_VALID_CLAIM_TYPE`, `_VALID_CLAIM_LEVEL`, `_VALID_MATERIALITY`, `_VALID_SECTION_KEY`, `_VALID_SOURCE_TYPE`, `_VALID_CONFIDENCE`
- Also fixed: `pipeline` was in prompt but missing from model choices → added `pipeline` to `AIClaim.section_key choices` (models/reports/__init__.py) and `_VALID_SECTION_KEY`

### Temporal Validation — ROC Year Misidentification (Fixed)
- Root cause: prompt had no ROC→西元 conversion rule; Claude mapped 民國115年 → 2025 instead of 2026 (115 + 1911 = 2026)
- Symptom: "時間軸不一致：使用者指定2026Q1，但文件實際期間為2025Q1（民國115年1月1日至3月31日）"
- Fix (`prompts/__init__.py`): added explicit conversion table in Phase 1 header:
  - 民國113年=2024, 民國114年=2025, 民國115年=2026, 民國116年=2027
  - rule: document_period must be converted to 西元年 before output
