## Root Cause Analysis：Memory 漏更新問題

**問題**：每次 session 結束後 memory 沒有自動更新，需要人工提醒才更新。

**為什麼 AI governance 沒有阻止這件事**：

1. **AGENTS.md 是文字規則，不是執行約束**
   - AGENTS.md 第 263-283 行明確定義了 "Memory Update Required After Each Phase" 流程
   - 但這只是 prompt-layer 指令，依賴 AI 在 session 中自發遵守
   - 沒有工具層或 hook 層強制執行

2. **DoD (Definition Of Done) 沒有 fail-closed 機制**
   - AGENTS.md 第 94-101 行定義了 DoD：memory + PLAN.md + commit + push
   - 引用了 `scripts/closeout.ps1` 做 DoD enforcement
   - **但 `scripts/closeout.ps1` 不存在**，引用從未建立

3. **Session 壓縮後 context 遺失**
   - 長對話被壓縮後，AI 從 summary 重啟
   - summary 不包含「我還沒更新 memory」這種中間狀態
   - AI 繼續工作，不知道 DoD 未完成

4. **沒有 git pre-push hook**
   - 沒有 hook 阻止在 memory 未更新時 push
   - push 成功了，AI 認為任務完成

**修正方向**（待 Phase 10 或獨立修正）：
- ✅ `scripts/closeout.ps1` 已建立（從 ai-governance-framework 複製）
- ✅ `.git/hooks/pre-push` 已安裝新版（fail-closed，攔截無 memory 更新的 push）
- Hook 驗證：valid=True，pre-push 正常攔截
